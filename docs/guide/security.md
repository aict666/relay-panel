# 安全配置

RelayPanel 会控制公网监听端口和转发节点，建议把下面项目作为上线前的最低要求。

## 管理员账号

- 初始管理员密码由面板随机生成，只在首次初始化日志中输出一次；使用 `docker compose logs panel` 查看。
- 限制 Docker/容器日志的读取权限，并在首次登录后立即改成唯一的强密码。
- 不要复用服务器 SSH、邮箱或其他面板密码。
- 只把管理员权限授予确实需要的人；普通使用者使用普通账号和套餐授权。

## HTTPS 与反向代理

公网面板应使用 HTTPS。推荐在面板前使用 Caddy、Nginx 或其他反向代理，并确保 WebSocket 升级头正确传递。完整示例见[反向代理](/REVERSE-PROXY)和[简化 TLS 部署](/TLS-SIMPLE)。

反代后至少验证：

```bash
curl -I https://你的面板域名/
```

然后登录面板，确认节点日志出现 `websocket connected`。WebSocket 不通时 HTTP 轮询仍可工作，但配置更新会延迟。

## 防火墙最小开放

- 面板：只开放 HTTP/HTTPS 对外端口，数据库端口不要暴露公网。
- 节点：只开放实际使用的入口端口和设备组端口池。
- SSH：限制来源地址或使用 VPN/堡垒机，禁用密码登录更稳妥。
- 多跳：尽量只允许上一跳节点访问下一跳端口池。

`tcp_udp` 的 UDP 多跳使用 UOT 时会额外占用端口池内的 TCP 端口。只开放原始 UDP 端口会导致 UOT 隧道失败。

预设隧道的 TCP、UDP 和 `tcp_udp` 规则统一连接防火墙清单中的共享 **TCP** 端口。公网入口端口与共享内部端口用途不同；内部端口应尽量只允许上一跳节点地址访问。

## Token 保护与轮换

设备分组 token 等同于节点接入凭据：

- 不要提交到 Git 仓库、公开日志、Issue 或截图。
- 节点上的 `/opt/relay-node/start.sh` 应保持仅 root 可读。
- 怀疑泄露时，在 **设备分组** 中重新生成 token，并重新执行各节点的对接命令。

UOT 使用面板派生的独立 HMAC 密钥做相邻节点认证，静态 token 不直接在链路上传输；但 UOT **不加密业务 payload**。敏感业务仍应使用 TLS、WireGuard、QUIC、Shadowsocks 等端到端加密。

预设隧道也只提供相邻节点的 HMAC 身份认证和握手重放防护，业务 payload 不加密、也不做逐包完整性校验。认证头带有受 HMAC 保护的时间戳，允许最多 60 秒系统时钟偏差；所有节点必须启用 NTP/chrony 等时间同步。链路中间人仍可能观察或修改载荷，因此敏感业务必须使用端到端 TLS、WireGuard 等保护。轮换设备组 token 会撤销派生密钥并关闭受影响的旧隧道连接；即使 WebSocket 不可用，HTTP 配置轮询收到凭据拒绝后也会立即停止缓存转发并关闭旧连接。撤销通知触发的新配置暂时不可用时，节点也会把清理后的 fail-closed 状态写入本地缓存，避免重启恢复旧密钥。仍配置旧 token 的节点安装新 token 后才能恢复。

## 预设隧道权限

新建和升级得到的预设隧道默认是“管理员专用”。只有管理员主动开启“允许普通用户使用”，并且用户套餐已授权该隧道的入口设备组时，普通用户才会在规则表单中看到并可绑定它。服务端会同时在目录接口、规则写事务和节点配置生成三层复核，直接提交或猜测 `tunnel_id` 不会绕过授权。

自定义多跳与预设隧道的授权边界不同：普通用户可以自行决定自定义链的拓扑，因此入口、中继和出口每一跳都必须在其套餐授权内；预设隧道的内部拓扑由管理员控制，所以普通用户只需获得已共享隧道的入口授权。套餐撤销后，含未授权 hop 的存量自定义链也会停止向节点下发。

关闭共享或撤销入口线路授权会立即停止向节点下发相关普通用户规则；管理员规则不受影响。内部共享端口本身不是用户权限边界，不要因为多条规则复用一个端口就把该端口向整个公网开放，仍应仅允许上一跳节点访问。

## 备份

升级和重大配置变更前备份：

- `/opt/relay-panel/.env`
- SQLite 数据库文件，或 PostgreSQL 的一致性备份
- 当前使用的 Compose 和反代配置

备份应存放在不同磁盘或不同机器，并定期做一次恢复演练。只有“能恢复”的备份才有意义。
