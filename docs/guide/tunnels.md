# 预设隧道与内部端口复用

预设隧道由管理员维护一条完整线路，规则只引用线路，不把业务目标写进隧道：

```text
客户端 → 规则公网入口端口 → 共享隧道端口 → 中继… → 出口 → 规则目标
```

这一区分很重要：

- **隧道**保存入口、中继和出口设备组，以及非入口 hop 的共享 TCP 端口。
- **规则**保存公网入口端口、TCP/UDP 协议、最终目标、负载策略、限速和连接数限制。
- 同一隧道上的多条规则，在每个非入口 hop 只占一个内部 TCP 端口；每条规则的公网入口端口仍然独占。

该表达参考了 RelayX 的[入口/出口隧道](https://docs.relayx.cc/guide/getting-started/tunnel/)和[多跳隧道链](https://docs.relayx.cc/guide/advanced/tunnel-chain/)概念，但 RelayPanel v1 的端口复用是“共享监听端口 + 每连接规则路由握手”，不是跨规则共用一条长期底层连接的 MTCP/MWSS 多路复用。

## 创建隧道

进入 **隧道管理 → 创建隧道**：

1. 填写唯一名称，选择是否立即启用，以及是否允许普通用户使用。
2. 按流量顺序添加 2–8 个不重复设备组。
3. 第一个 hop 必须支持入口；后续 hop 不能是监控组，并且必须有 `connect_host`。
4. 后续 hop 可让面板从设备组端口池自动分配 TCP 端口，也可指定固定的 `1–65535` 端口。
5. 保存后按防火墙清单，在对应设备组节点放行列出的 **TCP** 端口。

编辑路径时，未变化的设备组保留原共享端口。停用隧道会立即撤下入口规则和共享 listener，但端口预留仍保留；重新启用后，未被人工暂停且用户状态有效的规则会自动恢复。

## 共享与使用权限

“共享内部端口”和“允许普通用户使用”是两件事：

- **内部端口复用**始终生效。同一预设隧道的多条规则在非入口 hop 共用一个 TCP 监听端口，不因共享开关改变。
- **用户共享**控制谁能在规则中选择这条隧道。新建隧道默认是“管理员专用”，普通用户既看不到，也不能通过猜测 `tunnel_id` 越权绑定。

开启“允许普通用户使用”后，不需要再维护一套隧道用户列表。面板沿用现有的套餐线路授权：只有套餐已经授权该隧道**入口设备组**的用户才能看到和绑定，管理员始终可用。`授权全部设备组` 同样覆盖所有共享隧道的入口。

关闭共享会立即撤下该隧道上普通用户规则的公网入口和内部路由，并终止对应活跃连接；管理员规则不受影响。面板不会修改这些规则的人工 `paused` 状态，重新共享后，仍有入口授权且未被人工暂停的规则会自动恢复。套餐撤销入口授权时也会停止下发，避免历史绑定在授权变化后继续工作。

## 在规则中使用

规则的线路模式有三种：

| 模式 | 线路来源 | 内部端口 |
|---|---|---|
| 直连 | 入口直接访问目标 | 无额外 hop |
| 自定义多跳 | 规则自己保存 hop 列表 | 每条规则单独分配 |
| 预设隧道 | 引用管理员维护的隧道 | 同一隧道的规则共享 |

选择“预设隧道”后，hop 编辑器变为只读路径。规则的目标、负载策略、限速、协议和公网入口端口仍照常填写。停用的隧道仍会显示在已有规则上，但不能绑定新规则；管理员专用隧道也会保留在原规则记录中，但普通用户不能启动或重新绑定。

极简规则导入/导出不会携带 `tunnel_id`。数据库 ID 只在当前面板内有意义，跨面板自动绑定相同数字可能把流量导向错误线路。

## TCP、UDP 与认证握手

TCP、UDP、`tcp_udp` 都连接同一个内部 TCP 隧道端口。每条 TCP 连接或每条 UDP 暖通道建立时，入口先发送固定长度认证头，其中包含协议版本、隧道 ID、规则 ID、模式、Unix 时间戳、随机 nonce 和 HMAC：

- HMAC 密钥由相邻设备组 token、隧道 ID 与 hop 位置派生，不在链路上传输。
- 节点拒绝错误 HMAC、超出 60 秒时钟偏差、重复 nonce、未知规则、已停用规则和不匹配的 TCP/UDP 模式；所有 relay-node 应启用 NTP/chrony 等系统时间同步。
- 最终目标只来自面板配置，数据面调用方不能在认证头中指定地址。
- HMAC 仅在建立连接/暖通道时计算，认证后 TCP 恢复原始字节流；无限速 Linux 路径继续使用 `splice`。

UDP 沿用 UOT datagram 帧。一个规则的多个 UDP 会话共享自己的暖通道，不同规则不共享底层连接，但都会连接同一个隧道端口。

## 变更与诊断

- 修改名称不改变节点配置，也不会重启数据面。
- 修改目标或负载策略只影响新连接；已有 TCP 连接继续使用原目标直到关闭。
- 修改路径或端口后，新连接走新路径；已有 TCP 连接自然排空，UDP 暖通道重连。
- 轮换设备组 token 属于撤销凭据，节点会关闭受影响的旧连接；使用该分组凭据接入面板的节点会失败关闭，必须安装管理员刚生成的新 token 后才恢复转发。
- “重启规则”只重启该规则的公网入口 listener，不会重启共享端口上的其他规则。
- TCP 与 `tcp_udp` 诊断会发送认证 `probe`，逐 hop 验证共享 listener、规则路由和最终目标；纯 UDP 仍不支持诊断。
- 隧道管理中的“诊断”会选择该隧道上一条正在运行的 TCP 或 `tcp_udp` 规则作为安全探针，因此隧道必须已启用并至少有一条符合条件的绑定规则；探测地址仍只来自面板配置。

## 删除保护

仍被规则引用的隧道不能删除。被隧道路径引用的设备组也不能删除。修改已被使用的隧道入口时，面板会原子检查所有普通规则所有者对新入口的授权；存在未授权用户时整次更新回滚。
